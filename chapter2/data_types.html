Rust Programming Tutorial (Thai)

Data Types

ชนิดข้อมูล (Data Types) ใน Rust เป็นพื้นฐานสำคัญในการพัฒนาโปรแกรม การทำความเข้าใจชนิดข้อมูลต่างๆ จะช่วยให้เราเขียนโค้ดได้อย่างมีประสิทธิภาพและปลอดภัย ในบทนี้เราจะมาทำความรู้จักกับชนิดข้อมูลพื้นฐานใน Rust และวิธีการใช้งาน

ชนิดข้อมูลพื้นฐาน (Primitive Data Types)

Rust มีชนิดข้อมูลพื้นฐานหลายชนิดที่สามารถใช้ได้ตามความเหมาะสม ได้แก่:

1. ตัวเลขจำนวนเต็ม (Integer)

ตัวเลขจำนวนเต็มใน Rust มีหลายขนาดและสามารถเป็นทั้งเลขบวกและเลขลบ (signed) หรือเลขบวกอย่างเดียว (unsigned) โดยขนาดของตัวเลขจะเป็นไปตามจำนวนบิต เช่น i8, i16, i32, i64, i128 สำหรับ signed integers และ u8, u16, u32, u64, u128 สำหรับ unsigned integers

ตัวอย่างการใช้งาน:

#![allow(unused)]
fn main() {
let a: i32 = 42; // ตัวเลขจำนวนเต็มแบบ signed ขนาด 32 บิต
let b: u8 = 255; // ตัวเลขจำนวนเต็มแบบ unsigned ขนาด 8 บิต
}

2. ตัวเลขทศนิยม (Floating-Point)

Rust สนับสนุนตัวเลขทศนิยมที่มีขนาด 32 บิต (f32) และ 64 บิต (f64) ซึ่ง f64 เป็นค่าเริ่มต้น

ตัวอย่างการใช้งาน:

#![allow(unused)]
fn main() {
let x: f64 = 3.14; // ตัวเลขทศนิยมขนาด 64 บิต
let y: f32 = 2.5; // ตัวเลขทศนิยมขนาด 32 บิต
}

3. ตัวอักษร (Character)

Rust ใช้ char สำหรับเก็บตัวอักษร ซึ่งเป็นชนิดข้อมูลที่สามารถเก็บ Unicode scalar value ได้

ตัวอย่างการใช้งาน:

#![allow(unused)]
fn main() {
let c: char = 'A'; // ตัวอักษร
let heart_eyed_cat: char = '😻'; // อีโมจิ
}

4. บูลีน (Boolean)

Rust ใช้ bool สำหรับเก็บค่าทางตรรกะที่เป็นจริง (true) หรือเท็จ (false)

ตัวอย่างการใช้งาน:

#![allow(unused)]
fn main() {
let t: bool = true; // ค่า true
let f: bool = false; // ค่า false
}

5. สตริง (String)

Rust มีชนิดข้อมูลสตริงสองแบบ คือ &str (string slice) และ String ซึ่ง &str เป็นชนิดข้อมูลที่มีประสิทธิภาพสูงและใช้สำหรับการอ้างอิงสตริงแบบคงที่ ในขณะที่ String เป็นสตริงที่สามารถเปลี่ยนแปลงค่าได้

ตัวอย่างการใช้งาน:

#![allow(unused)]
fn main() {
let s1: &str = "Hello, world!"; // string slice
let mut s2: String = String::from("Hello, Rust!"); // String
s2.push_str(" Welcome!"); // เพิ่มข้อความใน String
}

ความแตกต่างระหว่าง &str และ String

&str เป็น string slice ที่ใช้อ้างอิงสตริงแบบคงที่และมีประสิทธิภาพสูง เนื่องจากอยู่บน Stack และไม่สามารถเปลี่ยนแปลงค่าได้

String เป็นชนิดข้อมูลที่ใช้เก็บสตริงที่สามารถเปลี่ยนแปลงค่าได้ และถูกเก็บใน Heap ซึ่งมีความยืดหยุ่นในการจัดการหน่วยความจำ

ข้อดีของ &str:

ประสิทธิภาพสูง เนื่องจากอยู่บน Stack เหมาะสำหรับค่าคงที่ ข้อดีของ String:

ยืดหยุ่น สามารถเปลี่ยนแปลงค่าได้ เหมาะสำหรับการจัดการสตริงที่มีการเปลี่ยนแปลง

การจัดการหน่วยความจำ: Stack และ Heap

Stack

Stack เป็นพื้นที่หน่วยความจำที่ใช้เก็บข้อมูลชั่วคราวเช่นตัวแปรที่ประกาศในฟังก์ชัน ข้อมูลที่เก็บใน Stack จะถูกจัดการแบบอัตโนมัติและมีการเข้าถึงที่รวดเร็ว แต่ขนาดของ Stack จะถูกจำกัด

ตัวแปรที่เก็บใน Stack มักจะเป็นตัวแปรที่มีขนาดเล็กและมีอายุการใช้งานสั้น เช่น:

fn main() {
    let a = 10; // ตัวแปร a ถูกเก็บใน Stack
    let b = 20;
    println!("a: {}, b: {}", a, b);
}

Heap

Heap เป็นพื้นที่หน่วยความจำที่ใช้เก็บข้อมูลที่มีขนาดใหญ่หรือมีอายุการใช้งานยาวนาน ข้อมูลใน Heap จะต้องจัดการด้วยตนเอง การเข้าถึงข้อมูลใน Heap จะช้ากว่า Stack แต่สามารถเก็บข้อมูลได้มากกว่า

ตัวอย่างการใช้ Heap ใน Rust:

fn main() {
    let s1 = String::from("Hello, world!"); // ตัวแปร s1 ถูกเก็บใน Heap
    println!("{}", s1);
}

ในตัวอย่างนี้ สตริง s1 ถูกเก็บใน Heap เนื่องจากมีขนาดใหญ่และมีการจัดการหน่วยความจำโดย Rust

ความสำคัญของ Stack และ Heap ใน Rust

การจัดการหน่วยความจำใน Rust นั้นพิเศษตรงที่ Rust ไม่มี garbage collector แบบภาษาโปรแกรมระดับสูงอื่น ๆ เช่น Java หรือ Python การจัดการหน่วยความจำใน Rust ทำได้โดยใช้หลักการ Ownership, Borrowing และ Lifetimes ซึ่งทำให้การจัดการหน่วยความจำมีประสิทธิภาพและปลอดภัย

Stack: ข้อมูลที่เก็บใน Stack จะถูกจัดการแบบอัตโนมัติ มีประสิทธิภาพในการเข้าถึงสูง แต่ขนาดของ Stack จะถูกจำกัด Heap: ข้อมูลที่เก็บใน Heap จะต้องจัดการด้วยตนเอง แต่มีความยืดหยุ่นในการจัดการหน่วยความจำ สามารถเก็บข้อมูลได้มากกว่าและใช้งานได้นานกว่า

Ownership, Borrowing และ Lifetimes

Ownership

ใน Rust ทุกค่ามีตัวเจ้าของ (owner) และในเวลาใดๆ จะมีเพียงเจ้าของเดียวเท่านั้น เมื่อเจ้าของหมดอายุ (ออกนอกขอบเขตการใช้งาน) ค่านั้นจะถูกปล่อยคืนหน่วยความจำ (dropped) การใช้ Ownership ช่วยให้ Rust จัดการหน่วยความจำได้โดยไม่ต้องใช้ garbage collector

ตัวอย่างการใช้งาน Ownership:

fn main() {
    let s1 = String::from("Hello");
    let s2 = s1; // Ownership ของ "Hello" ย้ายจาก s1 ไปยัง s2
    // println!("{}", s1); // จะเกิดข้อผิดพลาดเนื่องจาก s1 ไม่มีค่าอีกต่อไป
}

Borrowing

Borrowing คือการยืมค่ามาใช้งานชั่วคราวโดยไม่โอน Ownership การ Borrowing แบ่งออกเป็นสองประเภท คือ ยืมแบบอ่านอย่างเดียว (immutable borrow) และยืมแบบเปลี่ยนแปลงค่าได้ (mutable borrow)

ตัวอย่างการใช้งาน Borrowing:

fn main() {
    let s1 = String::from("Hello");
    let len = calculate_length(&s1); // ยืม s1 มาใช้งาน
    println!("The length of '{}' is {}", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}

Lifetimes

Lifetimes ใช้เพื่อระบุขอบเขตการใช้งานของการยืมค่า ช่วยให้ Rust ตรวจสอบและป้องกันข้อผิดพลาดจากการอ้างอิงค่าที่หมดอายุแล้ว

ตัวอย่างการใช้งาน Lifetimes:

fn main() {
    let r;
    {
        let x = 5;
        r = &x; // Error: x หมดอายุการใช้งานก่อน r
    }
    println!("r: {}", r);
}

การใช้ Lifetimes ช่วยให้การจัดการหน่วยความจำใน Rust ปลอดภัยและปราศจากข้อผิดพลาด

การเลือกใช้ชนิดข้อมูลและการจัดการหน่วยความจำ

การเลือกใช้ชนิดข้อมูลที่เหมาะสมเป็นสิ่งสำคัญในการพัฒนาโปรแกรมที่มีประสิทธิภาพและปลอดภัย ข้อควรระวังในการใช้ชนิดข้อมูลได้แก่:

การใช้ขนาดของตัวเลขที่เหมาะสม: เลือกใช้ขนาดของตัวเลขที่พอดีกับข้อมูลที่เก็บ เพื่อประหยัดหน่วยความจำและเพิ่มประสิทธิภาพ การตรวจสอบขอบเขตของตัวเลข: ระมัดระวังในการใช้ตัวเลขที่มีขอบเขตจำกัด เช่น u8 เพื่อหลีกเลี่ยงการล้นของตัวเลข (overflow) การใช้ชนิดข้อมูลที่ตรงกับความต้องการ: เช่น ใช้ bool สำหรับค่าทางตรรกะ และ char สำหรับตัวอักษร

สรุป

ชนิดข้อมูลใน Rust เป็นพื้นฐานสำคัญในการพัฒนาโปรแกรม Rust มีชนิดข้อมูลพื้นฐานหลายชนิดที่สามารถเลือกใช้ได้ตามความเหมาะสม การทำความเข้าใจและใช้ชนิดข้อมูลที่ถูกต้องจะช่วยให้โปรแกรมของเรามีประสิทธิภาพและปลอดภัยมากยิ่งขึ้น การจัดการหน่วยความจำใน Rust โดยใช้ Stack และ Heap ทำให้การเขียนโปรแกรมมีความยืดหยุ่นและมีประสิทธิภาพมากยิ่งขึ้น รวมถึงการเข้าใจหลักการ Ownership, Borrowing และ Lifetimes ที่เป็นเอกลักษณ์ของ Rust จะช่วยให้การพัฒนาโปรแกรมมีความปลอดภัยและปราศจากข้อผิดพลาด
